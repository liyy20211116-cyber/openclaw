# OpenClaw + 飞书 P4 联调与灰度清单

## A. 联调前检查（5 分钟）

- [ ] 网关已启动：`npx openclaw gateway --port 18789 --verbose`
- [ ] 通道状态正常：`npx openclaw channels status --probe`
- [ ] 预期状态包含：`Feishu default: enabled, configured, running, works`
- [ ] 模型状态正常：`npx openclaw models list` 可见 `openai-codex/gpt-5.3-codex`

## B. 功能联调（必须全部通过）

### B1 私聊回归

- [ ] 在飞书私聊机器人发送：`你好，返回当前时间`
- [ ] 收到回复，且时延可接受（建议 < 10 秒）
- [ ] 连续发 2~3 条，确认上下文连续

### B2 群聊触发策略

- [ ] 群聊不 `@机器人` 发送消息（应按策略不触发）
- [ ] 群聊 `@机器人` 发送消息（应触发并回复）
- [ ] 群聊里连续对话 2 轮，确认不串会话

### B3 异常路径

- [ ] 人工输入超长消息，确认系统可处理或给出合理错误提示
- [ ] 人工断网后恢复，确认网关可自动恢复连接

## C. 日志核查（抽样）

- [ ] 日志存在入站消息记录
- [ ] 日志存在模型调用记录
- [ ] 日志不存在持续 4xx/5xx 风暴
- [ ] 日志不存在凭据明文泄漏

## D. 灰度策略（24 小时）

- [ ] 仅开放 1 个用户 + 1 个群
- [ ] 观察错误率、响应时延、消息重复率
- [ ] 观察 token 消耗是否符合预期
- [ ] 无高频异常后再逐步放量

## E. 验收标准（P4 通过条件）

- [ ] 私聊与群聊主路径稳定可用
- [ ] 触发策略符合预期（特别是 @ 触发）
- [ ] 24 小时内无阻断性故障
- [ ] 可定位问题并有回滚手段

## F. 回滚预案（出问题立即执行）

1. 暂时关闭飞书事件接入或移出机器人
2. 保留网关日志并记录最后 50 行错误
3. 恢复到上一个可用配置快照
4. 重新启动网关并验证 `channels status --probe`
